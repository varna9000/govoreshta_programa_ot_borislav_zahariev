# Говореща програма — MicroPython

**Порт на речевия синтезатор на Борислав Захариев за микроконтролери**

Оригиналната „Говореща програма" е създадена в края на 80-те години за компютъра Правец 8 (клонинг на Apple II). Този порт я пренася на MicroPython, като запазва оригиналния метод на синтез — директно превключване на говорител с точно контролирано време между импулсите.

Налични са два варианта:

| Файл | Платформа | Метод |
|------|-----------|-------|
| `govor_rp2040_pio.py` | Raspberry Pi Pico / RP2040 | PIO — тактово точен, нулев джитър |
| `govor_micropython.py` | ESP32, ESP8266, STM32, Pico | PWM + `sleep_us()` — универсален |

## Бърз старт

```python
from govor import Govor

g = Govor(pin=15)
g.speak("zdrasti")
g.say("здрасти")
g.demo()               # "говореща програма от борислав захариев"
```

## Свързване

### Най-просто — директно от GPIO

Работи веднага, но звукът е тих (GPIO може да даде само ~12 mA).

```
GPIO ────[ 100Ω ]────( говорител + )
                      ( говорител − )──── GND
```

Може да се използва и пиезо-зумер вместо говорител — той е по-силен при директно свързване.

### По-силно — с транзистор

NPN транзистор (2N2222, BC547, BC337 или подобен) усилва тока. Говорителят се захранва от 5V вместо от GPIO, което дава значително по-силен звук.

```
                         5V (VBUS на Pico, VIN на ESP32)
                          │
                     ( говорител + )
                          │
                     ( говорител − )
                          │
                      Колектор (C)
GPIO ────[ 1kΩ ]──── База (B)           NPN транзистор
                      Емитер (E)         (2N2222, BC547)
                          │
                         GND
```

### Най-силно — с усилвател LM386

Модул с LM386 (струва ~1 лв от AliExpress) дава истински силен звук с възможност за регулиране на силата.

```
GPIO ────[ 10kΩ потенциометър ]──── IN на LM386
                                    OUT ──── говорител
                                    VCC ──── 5V
                                    GND ──── GND
```

### По-чист звук — с RC филтър

Аудио сигналът е правоъгълна вълна. Нискочестотен RC филтър изглажда острите ръбове и намалява шума. Може да се добави към всяка от горните схеми, между изхода и говорителя.

```
Изход ────[ 1kΩ ]────┬──── към говорител
                      │
                   [100nF]
                      │
                     GND
```

## Вариант 1: RP2040 PIO (Raspberry Pi Pico)

**Файл:** `govor_rp2040_pio.py` → преименувайте на `govor.py` на платката.

Този вариант използва PIO (Programmable I/O) на RP2040 — хардуерна подсистема, която изпълнява собствена програма от 8 инструкции, независимо от процесора. PIO тактовата честота е зададена на **1 023 000 Hz** — точно колкото процесорът на Apple II. Една PIO инструкция = един 6502 такт.

```
┌─────────────┐         ┌──────────────────┐
│  Python     │  FIFO   │  PIO програма    │
│  подава     │ ──────► │  8 инструкции    │──── GPIO ──── говорител
│  стойности  │  8 дълб.│  тактово точни   │
└─────────────┘         └──────────────────┘
```

Python изпраща стойности за закъснение във FIFO буфера (8 позиции дълбок). PIO ги чете, превключва пина и брои точния брой тактове. Докато FIFO-то не е празно, звукът е абсолютно без джитър — целият Python overhead се поглъща от буфера.

### Характеристики

- Джитър: **0 такта** (PIO е детерминистичен)
- Натоварване на CPU: **~0%** (PIO работи независимо)
- Тактова честота: **1.023 MHz** (= Apple II)
- RAM: **~11 KB** (4.6 KB фонеми + 1 KB код + overhead)

### Примерно свързване за Pico

```
              ┌─────────────────┐
              │  Raspberry Pi   │
              │     Pico        │
              │                 │
    говорител │ GP15 ──[100Ω]──┤──── (+)
              │ GND ────────────┤──── (−)
              │                 │
              └─────────────────┘
```

### Употреба

```python
from govor import Govor

g = Govor(pin=15)              # GP15
g.speak("zdrasti")             # латиница
g.say("здрасти")               # кирилица
g.say("добър ден как сте")
g.demo()                       # класическото представяне
g.speed = 5                    # по-бавно
g.speak("bawno goworene")
g.test()                       # бърз тест: гласни + поздрав
```

Може да се използва всеки GPIO пин и всяка свободна PIO машина (0–7):

```python
g = Govor(pin=22, sm_id=1)    # GP22, PIO SM1
```

## Вариант 2: Универсален (ESP32, ESP8266, STM32, Pico)

**Файл:** `govor_micropython.py` → преименувайте на `govor.py` на платката.

Работи на всяка платка с MicroPython. Използва PWM при 40 kHz (над звуковия диапазон) за по-силен ток към говорителя и `time.sleep_us()` за контрол на времето. Функциите за синтез са декорирани с `@micropython.native` за по-малък overhead.

### Характеристики

- Джитър: **±5–20 µs** (от Python интерпретатора)
- Натоварване на CPU: **100%** по време на говорене
- RAM: **~11 KB** (4.6 KB фонеми + 1 KB код + overhead)
- Качество: добро — речта е разпознаваема, с лек роботизиран характер (като оригинала)

### Примерно свързване за ESP32

```
              ┌─────────────────┐
              │     ESP32       │
              │                 │
    говорител │ GPIO15 ─[100Ω]─┤──── (+)
              │ GND ────────────┤──── (−)
              │                 │
              └─────────────────┘
```

За ESP8266 използвайте GPIO пин, който поддържа PWM (GPIO 0, 2, 4, 5, 12, 13, 14, 15).

### Употреба

Същата като PIO варианта:

```python
from govor import Govor

g = Govor(pin=15)
g.speak("zdrasti")
g.say("здрасти")
g.demo()
g.speed = 5
g.deinit()                     # освобождава PWM ресурса
```

## Как работи оригиналът

Правец 8 има 1-битов говорител, управляван от адрес `$C030`. Всеки запис на този адрес обръща мембраната на говорителя. Захариев кодира всяка фонема като таблица от байтове, задаващи времето между обръщанията — бързите обръщания дават висок тон, бавните — нисък. Комбинацията наподобява формантите на човешката реч.

Два режима на синтез:

- **Нибъл режим** (префикс `0xFF`): всеки байт се разделя на две 4-битови стойности — по една за всеки полупериод. Използва се за съгласни (с, ш, т, ч, ф, х…).
- **Байтов режим**: целият байт е време за един полупериод. Използва се за гласни (а, е, и, о, у…).

Целият синтезатор е ~6 KB машинен код за процесор 6502, съдържащ 30 български фонеми.

## Отстраняване на проблеми

**Няма звук:**
- Проверете дали пинът е правилен (`Govor(pin=15)` → GPIO 15)
- Проверете свързването на говорителя
- Опитайте с `g.test()` — трябва да чуете гласни и „здрасти"

**Много тих звук:**
- Нормално е при директно GPIO свързване — добавете транзисторен усилвател
- Пиезо-зумерът е по-силен от малък говорител при директно свързване
- Увеличете съпротивлението (или махнете резистора за тест, няма да повреди нищо)

**Звукът е шум (универсален вариант):**
- Намалете скоростта: `g.speed = 10` (по-бавно = по-стабилно)
- На ESP8266 е възможно `sleep_us()` да не е достатъчно точен — опитайте ESP32

**`MemoryError`:**
- Не трябва да се случва (буферът е ~6 KB), но ако се случи — рестартирайте платката и заредете само `govor.py`, без други модули

**PIO вариантът не работи:**
- Уверете се, че платката е RP2040 (Pico, Pico W, и др.)
- Проверете дали `sm_id` не е зает от друг PIO код
- Опитайте друга PIO машина: `Govor(pin=15, sm_id=2)`
